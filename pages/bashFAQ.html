Set -- Builtin Command

Here are some flags may not be set in shell script. Cause they may throw errorin some condition and not error in another condition

The use of this flags will have some side effect.

1.set -e

set -e was an attempt to add "automatic error detection" to the shell. Its goal was to cause the shell to abort any time an error occurred.

The following code will print anything, if we set -e

 
set -e
i=0
let i++      #will abort here, so there is nothing to print
echo "i is $i"

This script will print anything, cause the let manual say"If the last ARG evaluates to 0, let returns 1; 0 is returned otherwise." i++ evaluates to 0, so let i++ returns 1 and trips the set -e. The script aborts. Because we added 1 to a variable."

if you change the i into 1, they will print. But if you change let i++ into ((i++)). It only worked in bash4.0 and earily version. so not use that in script,unless you have this flag in particularly.
